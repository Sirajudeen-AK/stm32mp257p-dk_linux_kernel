# Program 5 - file->private_data and container_of()

## Problem

Global buffer means all users share state.

## Solution

file->private_data

open():
file->private_data = dev;

read():
dev = file->private_data;

## container_of()

Given:
struct mydev {
    int id;
    struct cdev cdev;
};

container_of(inode->i_cdev, struct mydev, cdev)

returns pointer to mydev.

## Interview Questions
Why not globals?
How multiple devices handled?


private_data + container_of()

Goal: Maintain separate device instances.

Flow:
open()
  inode->i_cdev
      |
container_of()
      |
my_device structure
      |
file->private_data = dev

Later:
read()/write()
  dev = file->private_data

Benefit:
Multiple minors can share the same driver while keeping separate buffers/state.
